결제 API 연동 문서

외부 연동 업체와 내부 운영팀이 함께 참고할 수 있도록, 현재 활성 결제 화면 /[lang]/[clientid]/[center]/payment 기준의 API 흐름과 서버 내부 오케스트레이션 범위를 상세하게 정리한 기술 문서입니다.

문서 범위

  • 브라우저 또는 외부 클라이언트가 직접 호출하는 Next.js API
  • Next.js API 가 내부적으로 호출하는 Stable 백엔드 API
  • 서버 내부에서만 호출해야 하는 Payaction, thirdweb 관련 처리

핵심 요약

  • 현재 결제 페이지에서는 스마트 어카운트 연결 없이도 구매 진행이 가능합니다.
  • 핵심 주문 생성 API 는 POST /api/order/setBuyOrder 입니다.
  • 결제요청은 서버가 https://api.payaction.app/order 를 대신 호출합니다.
  • 결제확정은 thirdweb SDK 또는 thirdweb engine 으로 처리됩니다.
권장 연동 방향

외부 업체는 가능하면 Next.js API 기준으로 연동하고, Stable 백엔드 직접 연동은 필요한 경우에만 제한적으로 허용하는 것을 권장합니다.

1

환경 조회

클라이언트 정보와 가맹점 정보를 읽어 결제 조건을 준비합니다.

2

구매자 준비

입금자 정보 기준으로 구매자 지갑을 생성하거나 기존 사용자 정보를 조회합니다.

3

기존 주문 확인

같은 사용자의 진행 중 주문이 있으면 새 주문을 만들지 않고 상세 화면으로 이동합니다.

4

매칭 시도

같은 KRW 금액의 오픈 판매 주문이 있으면 즉시 수락합니다.

5

구매 주문 생성

매칭 주문이 없으면 신규 구매 주문을 생성합니다.

6

후속 처리

주문 상세 페이지로 이동한 뒤 필요 시 결제요청과 결제확정을 진행합니다.

운영 메모

현재 활성 흐름에서는 주문 생성 후 pay-usdt-reverse/[orderId] 로 이동하며, 결제요청과 결제확정은 서버 내부 연동 범위로 보는 것이 적절합니다.

외부 업체가 알아야 하는 주요 API

실제 연동과 운영 대응에 필요한 API 들을 목적 중심으로 정리했습니다.
API 목적 실제 대상 주요 사용 포인트
POST /api/client/getClientInfo 클라이언트 환경 조회 POST {STABLE_API_URL5}/api/client/getClientInfo 체인, 환율 정보 사용
POST /api/store/getOneStore 가맹점 정보 조회 POST {STABLE_API_URL5}/api/store/getOneStore 운영 상태, agentcode, 결제 한도 확인

POST /api/order/getOneBuyOrderByNicknameAndStorecode 기존 진행 주문 확인 로컬 DB 진행 중 주문 재진입 처리

Next.js API 기준 연동 장점

  • 외부 업체가 내부 Stable 백엔드 구조를 알 필요가 없습니다.
  • Payaction, thirdweb 관련 비밀키를 노출하지 않습니다.
  • 감사로그, 예외처리, 후속 라우팅을 서버에 캡슐화할 수 있습니다.

직접 Stable 연동 시 주의

  • 주문 생성 외의 후속 결제 처리 범위를 별도 정의해야 합니다.
  • 인증 정책, 허용 IP, 서명 체계가 추가로 필요할 수 있습니다.
  • 응답 포맷이 API 별로 완전히 동일하지 않다는 점을 감안해야 합니다.

서버 내부 처리: 결제요청

POST /api/order/requestPayment 는 서버가 https://api.payaction.app/order 를 대신 호출하는 구조입니다.

Payaction Key 서버 보관 x-mall-id 서버 보관

서버 내부 처리: 결제확정

POST /api/order/confirmPayment 는 thirdweb SDK 또는 thirdweb engine API 로 에스크로 전송과 상태 확정을 수행합니다.

thirdweb SDK thirdweb engine

응답 형식 메모

  • 기본 성공 패턴은 { "result": {} } 형태입니다.
  • 실패 시 { "result": null, "error": "..." } 패턴을 주로 사용합니다.
  • 일부 API 는 walletAddress 같은 필드를 직접 반환합니다.

사전 확인 필요 항목

  • Stable 백엔드의 phoneNumber, mobile 저장 여부
  • returnUrl, orderNumber 포맷 규칙
  • 주문 상태값 전체 enum 목록
  • 외부 업체의 직접 호출 대상이 Next.js 인지 Stable 인지 여부
보안 체크리스트

Payaction API Key, Mall ID, THIRDWEB_SECRET_KEY, THIRDWEB_ENGINE_ACCESS_TOKEN, DB 접속 정보는 반드시 서버에서만 관리해야 합니다.
